Introduction

Matrix.css is a comprehensive CSS framework inspired by The Matrix aesthetic. It provides a dark theme with neon green accents, complete component library, and utility classes for building futuristic, cyberpunk-inspired interfaces.

The framework includes everything from basic UI components to advanced Matrix-themed effects, making it perfect for cybersecurity dashboards, sci-fi themed websites, or any project that wants to capture that distinctive Matrix feel.

Note: Matrix.css is designed as a standalone CSS framework, no JavaScript required for most components. However, some interactive components may benefit from JavaScript for full functionality.

Installation

You can quickly include Matrix.css in your project using one of the following methods:

Direct Download

Download the latest version of Matrix.css and include it in your HTML: 

<link rel="stylesheet" href="path/to/matrix.css">

CDN

Include Matrix.css directly from a CDN: 

<link rel="stylesheet" href="https://cdn.example.com/matrix-css/1.0/matrix.css">

NPM

Install via NPM: 

npm install matrix-css

Then import it in your JavaScript: 

import 'matrix-css/dist/matrix.css';

Containers

Matrix.css provides flexible container classes to help structure your content. The framework includes fixed-width and fluid containers.

Fixed Container

The .m-container class creates a responsive fixed-width container that centers your content on the page. It has different max-widths at different breakpoints. 

.m-container {
  width: 100%;
  max-width: var(--m-container-width);
  margin: 0 auto;
  padding: 0 var(--m-space-md);
}

Fluid Container

The .m-container-fluid class creates a full-width container, spanning the entire width of the viewport. It maintains padding on the sides for content. 

.m-container-fluid {
  width: 100%;
  padding: 0 var(--m-space-md);
}

Spacing

Matrix.css provides a comprehensive set of spacing utilities to help manage margins and padding consistently throughout your project.

Spacing Variables

:root {
  --m-space-xs: 0.25rem;  /* 4px */
  --m-space-sm: 0.5rem;   /* 8px */
  --m-space-md: 1rem;     /* 16px */
  --m-space-lg: 1.5rem;   /* 24px */
  --m-space-xl: 2.5rem;   /* 40px */
}

Margin Utilities

Margin utilities use the format .m-m{side}-{size} where side can be blank (all sides), t (top), r (right), b (bottom), l (left), x (left and right), or y (top and bottom); and size ranges from 0 to 5. 

.m-m-0 { margin: 0 !important; }
.m-mt-3 { margin-top: var(--m-space-md) !important; }
.m-mr-2 { margin-right: var(--m-space-sm) !important; }
.m-mb-5 { margin-bottom: var(--m-space-xl) !important; }
.m-ml-1 { margin-left: var(--m-space-xs) !important; }
.m-mx-4 { margin-left: var(--m-space-lg) !important; margin-right: var(--m-space-lg) !important; }
.m-my-2 { margin-top: var(--m-space-sm) !important; margin-bottom: var(--m-space-sm) !important; }

Padding Utilities

Padding utilities follow the same pattern as margin utilities but with .m-p prefix. 

.m-p-0 { padding: 0 !important; }

.m-pr-2 { padding-right: var(--m-space-sm) !important; }

Glitch Text

The Glitch Text component creates a digital distortion effect that's perfect for emphasizing text in a Matrix-inspired interface. It simulates the "glitching" seen in the Matrix when reality breaks down.

Basic Glitch Text

System Malfunction

The glitch text effect creates a distorted, cyberpunk appearance that makes text look like it's experiencing a digital malfunction. The effect uses CSS pseudo-elements and animations to create the distortion. 

<h3 class="m-glitch-text" data-text="System Malfunction">System Malfunction</h3>

Glitch Text Implementation

The glitch effect works by creating two pseudo-elements that replicate the text content, then applying different animations and clip paths to each one. The data-text attribute must match the text content for the effect to work properly. 

.m-glitch-text {
  position: relative;
  color: var(--m-text);
  font-family: var(--m-font-main);
  letter-spacing: 2px;
}

.m-glitch-text::before,
.m-glitch-text::after {
  content: attr(data-text);
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: var(--m-bg);
}

.m-glitch-text::before {
  left: 2px;
  text-shadow: -2px 0 var(--m-danger);
  animation: m-glitch-text-1 2s infinite linear alternate-reverse;
  clip-path: polygon(0 0, 100% 0, 100% 45%, 0 45%);
}

.m-glitch-text::after {
  left: -2px;
  text-shadow: 2px 0 var(--m-info);
  animation: m-glitch-text-2 3s infinite linear alternate-reverse;
  clip-path: polygon(0 55%, 100% 55%, 100% 100%, 0 100%);
}

@keyframes m-glitch-text-1 {
  0% { transform: translateX(-2px); }
  100% { transform: translateX(2px); }
}

@keyframes m-glitch-text-2 {
  0% { transform: translateX(2px); }
  100% { transform: translateX(-2px); }
}

Responsive Design

Matrix.css is built with responsive design in mind, providing tools and utilities to create interfaces that work well on all screen sizes.

Responsive Breakpoints

Matrix.css uses standard breakpoints for responsive design, allowing you to create layouts that adapt to different screen sizes. 

/* Small devices (phones, 576px and up) */
@media (min-width: 576px) {
  /* sm classes and styles */
}

/* Medium devices (tablets, 768px and up) */
@media (min-width: 768px) {
  /* md classes and styles */
}

/* Large devices (desktops, 992px and up) */
@media (min-width: 992px) {
  /* lg classes and styles */
}

/* Extra large devices (large desktops, 1200px and up) */
@media (min-width: 1200px) {
  /* xl classes and styles */
}

/* Hide on all screen sizes */
